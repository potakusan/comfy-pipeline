# ComfyPipeline

ローカルの ComfyUI に接続し、**LoRA ごとの量産・吟味ワークフロー**を効率化するための Web UI です。  
シーン・ポーズ・追加プロンプトなどの微細な変化を与えた画像を大量生成し、その場で比較・選別できます。

---

## 主な用途

- 特定の LoRA に対して複数のシーン・ポーズ・構図パターンを一括キューに流し、大量生成する
- 追加プロンプトの「ランダム1行」モードで、1生成ごとに異なるタグを試しながらベストを探す
- ランダム構図モードで構図タグをシャッフルし、多彩なアングルの画像を効率よく吟味する
- 一括キュープリセットとして設定を保存しておき、次回は1クリックで同じ量産を再現する

---

## セットアップ

### 前提

- [ComfyUI](https://github.com/comfyanonymous/ComfyUI) がローカルで起動していること
- Node.js 18 以上、pnpm

### ComfyUI 側の設定

プレビュー画像をリアルタイム受信するため、ComfyUI の起動オプションに以下を追加してください。

```bat
--preview-method auto
```

### インストール

```bash
git clone <このリポジトリ>
cd comfy-pipeline
pnpm install
```

`.env.example` をコピーして `.env.local` を作成し、環境に合わせて編集します。

```bash
cp .env.example .env.local
```

```env
COMFYUI_URL=http://localhost:8188
NEXT_PUBLIC_COMFYUI_URL=http://localhost:8188

# ComfyUI の output ディレクトリへの絶対パス
# Windows: C:/Users/yourname/ComfyUI/output
# Mac/Linux: /home/yourname/ComfyUI/output
COMFYUI_OUTPUT_DIR=C:/path/to/ComfyUI/output
```

### 起動

```bash
pnpm dev
```

ブラウザで `http://localhost:3000` を開きます。

## 機能詳細

### LoRA 設定

可変 LoRA をリスト管理します。名前・強度・CLIP 強度・トリガーワードを設定し、生成時にリストから1つ選択して使用します。

### プロンプトビルダー

プリセットを組み合わせてポジティブプロンプトを構築します。

| カテゴリ   | 選択方式 | 用途                         |
| ---------- | -------- | ---------------------------- |
| 身体的特徴 | 複数可   | 髪色・体型など固定の外見特徴 |
| 人数       | 1択      | solo / 2girls など           |
| ポーズ     | 1択      | cowgirl position など        |
| シーン     | 1択      | 背景・場所                   |
| その他     | 複数可   | blush / embarrassed など     |

各プリセットには LoRA を紐付けることができ、そのプリセット選択時にシーン専用 LoRA を自動で適用できます。  
プリセットはドラッグ＆ドロップで並び替え可能です。

#### 追加プロンプト

テキストエリアに自由記述できます。モードは2種類：

- **全行使用** — テキストエリアの内容をそのまま毎回追加
- **ランダム1行** — 1行1タグとして入力し、生成ごとにランダムで1行を選択

ランダム1行モードでは、完成画像のサムネイルにカーソルを乗せると、その画像に実際に適用されたタグをプレビューと一緒に確認できます。

### サンプラー設定

チェックポイント・アップスケールモデル・解像度・ステップ数・CFG・サンプラー・スケジューラー・denoise を設定します。

解像度はプリセットボタンから素早く選択できます：

| プリセット | サイズ      |
| ---------- | ----------- |
| 横長       | 1536 × 1136 |
| 正方形     | 1024 × 1024 |
| 4:3        | 1536 × 1152 |
| 3:4        | 1152 × 1536 |
| 縦長       | 960 × 1600  |

### ランダム構図モード

有効にすると、各枚ごとにランダムな構図タグを1つ追加します。  
タグは1行1エントリで入力（例：`from above,` / `dutch angle,` / `close-up, upper body,`）。  
量産時に構図のバリエーションを自動で散らしたい場合に便利です。

### キューシステム

- **キューに追加** — 現在の設定でジョブを1件追加。「枚数」フィールドで1ジョブあたりの生成枚数を指定
- **中断** — 実行中のジョブを停止（次の pending ジョブは自動で継続）
- キューは順番に自動処理されます

### 一括キュープリセット

複数の設定パターンを「プリセットセット」として保存し、1クリックで全件キューに流す機能です。

#### 使い方

1. 左パネルで生成したい設定（シーン・ポーズ・追加プロンプトなど）を組む
2. **「一括キュー」** ボタンを開き、「新しいセットを作成」
3. 編集画面で **「現在の設定をプリセットとして追加」** を繰り返し、パターンを登録する  
   （例：カジノ×カウガール、ホテル×M字開脚、アウトドア×立ちポーズ…）
4. セットを保存
5. 次回以降は「一括キュー」→「実行」で、可変 LoRA を切り替えながら同じパターンを量産できる

プリセットごとに枚数・追加プロンプト・追加プロンプトモードを個別設定できます。  
セットはエクスポート/インポートで共有・バックアップできます。

### プレビューパネル

- 生成中はリアルタイムプレビューを常にメインビューに表示
- 生成完了した画像はストリップ（横並びサムネイル）に順番に追加
- **サムネイルにカーソルを乗せると大きなプレビューを表示**  
  ランダム1行モード使用時は適用されたタグもあわせて表示
- 生成完了後は最後の画像がメインビューに自動フォーカス  
  ストリップのサムネイルをクリックすると任意の画像をメインに切り替え可能

### ギャラリー

生成した画像の一覧。クリックで大きな画像を確認できます。  
「FS更新」で ComfyUI の output フォルダを直接スキャンして反映します。

### エクスポート / インポート

ヘッダーのボタンから設定を JSON ファイルとして出力・読み込みできます。  
以下の内容が含まれます：

- 可変 LoRA リスト
- 全プリセット（身体・人数・ポーズ・シーン・その他）
- サンプラー設定
- ネガティブプロンプト
- ランダム構図タグ
- 一括キュープリセットセット

## ワークフロー構成

ComfyUI に送信するワークフローは以下の構成で固定されています：

```
CheckpointLoaderSimple
  └─ LoraLoader × N  (FIXED_LORAS → プリセットLoRA → 可変LoRA の順)
       ├─ CLIPTextEncode (ポジティブ)
       ├─ CLIPTextEncode (ネガティブ)
       └─ KSampler
            └─ VAEDecode
                 └─ ImageUpscaleWithModel
                      └─ SaveImage
```

固定 LoRA（`FIXED_LORAS`）は常にチェーンの先頭に適用されます。  
カスタマイズする場合は [lib/comfy.ts](lib/comfy.ts) の `buildWorkflow` と `FIXED_LORAS` を編集してください。

## automosaic — 自動モザイクツール

画像内の性器等を YOLOv8 モデルで自動検出してモザイクをかけるツール。  
元スクリプト: [よくある質問 - sd_toshiaki Wiki](https://wikiwiki.jp/sd_toshiaki/%E3%82%88%E3%81%8F%E3%81%82%E3%82%8B%E8%B3%AA%E5%95%8F)

### 動作環境

- Windows + Python（PATH 通り済み）
- CUDA 対応 GPU 推奨（なければ CPU でも動作）
- 空き容量 6GB 以上

### セットアップ

#### 1. 初回起動（venv 作成 + 依存インストール）

`automosaic` ディレクトリで `automosaic.bat` をダブルクリックするか、画像ファイルをドロップします。  
初回は自動で venv が作成され pip インストールが走ります（数分かかります）。

```
[ERROR]処理対象の画像ファイルが見つかりませんでした
```

が表示されればインストール完了です。

#### 2. 検出モデルの配置

`automosaic/models/` フォルダにモデルファイルを配置します。

| 対象   | ファイル名   | 入手先                                                                                                                        |
| ------ | ------------ | ----------------------------------------------------------------------------------------------------------------------------- |
| 女性器 | `pussyV2.pt` | 自動ダウンロード（HuggingFace）、または [CivitAI](https://civitai.com/models/132388?modelVersionId=149617)                    |
| 男性器 | `penis.pt`   | [ntd11 モデル（CivitAI）](https://civitai.red/models/1313556?modelVersionId=2350456) からダウンロードし `penis.pt` にリネーム |
| 顔など | 任意の `.pt` | 別途 YOLOv8 セグメンテーションモデルを取得して `models/` に配置                                                               |

### 使い方

#### コマンドラインから

| 操作                                          | 説明                                                                   |
| --------------------------------------------- | ---------------------------------------------------------------------- |
| `automosaic.bat` をダブルクリック             | `input/` フォルダ内の画像を処理 → `output/` に出力（実行後に一時停止） |
| 画像 / フォルダを `automosaic.bat` にドロップ | 指定ファイルを処理 → `output/` に出力                                  |
| 画像を `automosaic_quick.bat` にドロップ      | 元画像と**同じフォルダ**に出力（プロンプト非表示）                     |

処理後のファイル名には `_mosaic` が付加されます。検出がなかった場合もファイルはそのままコピーして出力されます。

#### Web UI から使う（推奨）

`pnpm dev` で起動後、ヘッダーの **「画像処理」** リンク（`http://localhost:3000/process`）から GUI で操作できます。

```
┌─────────────────────────────────┬───────────────────────────────────┐
│  左パネル（設定）                │  右パネル（モニタリング）          │
│  - 処理対象フォルダ選択          │  システムリソース（常時表示）       │
│  - 自動モザイク設定              │    CPU% / GPU% / VRAM 折れ線グラフ │
│    モデル・クラス選択            │    2秒ごと自動更新（直近 80s）     │
│    信頼度 / モザイクサイズ       ├───────────────────────────────────┤
│    検知範囲拡張                  │  ジョブ進捗                        │
│  - リサイズ / 圧縮設定           │    進捗バー・枚数カウント          │
│    合計サイズ自動計算            │    ETA（経過 / 残り / 枚/秒）      │
│  - 処理開始ボタン                │    処理ログ                        │
│                                 │  処理前後ギャラリー                │
└─────────────────────────────────┴───────────────────────────────────┘
```

- **GPU グラフ**: NVIDIA GPU がある場合は GPU 使用率・VRAM も表示（`nvidia-smi` が PATH に必要）
- **ETA**: 2 枚以上処理されると処理速度（枚/秒）から残り時間を自動計算
- **処理順序**: リサイズ → モザイクの順で実行。大きな画像を先に縮小することで PNG 保存の I/O ボトルネックを解消

### オリジナル `automosaic.py` からの改善点

元スクリプトは動作・機能面でそのまま継承しつつ、以下を改善しています。

#### パフォーマンス

| 項目             | オリジナル                                     | 改善後                                                                                                                    |
| ---------------- | ---------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| モデルのロード   | 画像 1 枚ごとに毎回 `YOLO(path)` を呼び出す    | スレッドごとに初回のみロード（thread-local キャッシュ）                                                                   |
| 並列処理         | シングルスレッド逐次処理                       | `ThreadPoolExecutor` で並列処理（`-w` オプション）。PyTorch/CUDA は GIL を解放するためスレッド並列が有効                  |
| 検出前リサイズ   | 常にフルサイズで YOLO 推論                     | `--detect-size` で長辺を指定サイズに縮小してから推論。bbox / マスクは元サイズへスケールバックするので出力品質は変わらない |
| モザイク補間     | 縮小に `BICUBIC`（過平滑、モザイクらしくない） | 縮小に `BOX`（ダウンサンプリングに適した平均化補間）                                                                      |
| 処理時間の可視化 | なし                                           | `[検出] X.XXX秒 / [加工] X.XXX秒 / [保存] X.XXX秒` を各画像ごとに出力                                                     |

#### 機能追加

| オプション        | 説明                                                                                  |
| ----------------- | ------------------------------------------------------------------------------------- |
| `-w / --workers`  | 並列処理スレッド数（デフォルト: 1）。CUDA 環境では 2〜4 が目安                        |
| `--detect-size N` | 検出前に長辺を N px に縮小（例: `--detect-size 1024`）。大きな画像で推論を高速化      |
| `--classes`       | 検出クラスをカンマ区切りで絞り込み（`nipples,pussy,penis` 等）。元は全クラス固定      |
| `-e / --expand`   | 検知範囲を bbox サイズの指定割合で拡張。マスクも `MaxFilter` で同量拡張（元にはなし） |

#### 動作変更

| 項目                     | オリジナル                               | 改善後                                                  |
| ------------------------ | ---------------------------------------- | ------------------------------------------------------- |
| 検出なし時の出力         | ファイルを出力しない                     | オリジナルをそのままコピーして `_mosaic` ファイルを出力 |
| bbox の w/h チェック     | なし（ゼロサイズ bbox でエラーの可能性） | `w <= 0 or h <= 0` の場合スキップ                       |
| `combined_bboxes` 初期化 | `None` で初期化し `+= result.bboxes`     | 最初から空リストで `+=`（None チェック不要）            |

### リサイズ処理 (`resize.py`) の仕様

アスペクト比を保ったまま画像をリサイズし、メタデータ（Exif / PNG チャンク）を保持して保存するツール。

#### 基本仕様

| 項目              | 仕様                                                                 |
| ----------------- | -------------------------------------------------------------------- |
| 対応フォーマット  | JPEG / PNG / WebP / AVIF / BMP                                       |
| アスペクト比      | 保持（`LANCZOS` でリサンプリング）                                   |
| メタデータ        | JPEG / WebP / AVIF は Exif コピー、PNG は itxt チャンクコピー        |
| WebP ロスレス判定 | `VP8L` チャンクを検出して自動判定、ロスレスは `lossless=True` で保存 |
| スケール指定      | `0.01〜1.0`（例: `50` → 50%）                                        |
| 品質指定          | JPEG / WebP / AVIF の品質を 1〜100 で指定（PNG は無視）              |

#### 並列処理

- `-w / --workers` でスレッド数を指定（**デフォルト: CPU コア数**）
- PIL の `resize()` / `save()` は GIL を解放するため、スレッド並列が素直に効く
- PNG 圧縮は CPU バウンドなのでコア数に比例してスケール
- ディレクトリ作成の競合防止に `threading.Lock` を使用

#### Web UI での処理順序

```
入力フォルダ
    │
    ▼ (1) resize.py  ─ CPUコア数スレッドで並列リサイズ
_resize_tmp/ （中間ファイル）
    │
    ▼ (2) automosaic.py  ─ 縮小済み画像に YOLO 検出 + モザイク適用
{folder}/mosaic/  （最終出力）
    │
    ▼ _resize_tmp/ を自動削除
```

リサイズを先に行うことで、モザイク処理での **YOLO 前処理・モザイク適用・PNG 保存すべてが小さい画像で完結** します。元画像が 6400px の場合、50% リサイズ後は 3200px となり PNG 保存時間が数倍〜十倍程度改善します。

#### コマンドラインオプション

```
python resize.py <input_dir> [options]

-o DIR          出力フォルダ（デフォルト: output）
-s SCALE        リサイズ率 %（デフォルト: 50）
-q QUALITY      品質 1-100（JPEG/WebP/AVIF のみ、デフォルト: 85）
-w WORKERS      並列スレッド数（デフォルト: CPU コア数）
```

### automosaic コマンドラインオプション一覧

```
python automosaic.py [target ...] [options]

-m MODELS          使用モデルをカンマ区切りで指定（デフォルト: pussyV2.pt,penis.pt）
                   女性器のみ: -m pussyV2.pt
-o DIR             出力先フォルダを指定
-ssd               入力画像と同じフォルダに出力（automosaic_quick.bat はこれを使用）
-s SIZE            モザイクサイズ（デフォルト: 10、大きいほど粗い）
-c 0.25            信頼度しきい値（小さいほど広く検出、誤検知も増える）
-w N               並列処理スレッド数（デフォルト: 1）。CUDA 環境では 2〜4 が目安
--detect-size N    検出前に長辺を N px に縮小して推論、結果は元サイズにスケールバック
--classes CLASSES  検出クラスをカンマ区切りで指定（例: nipples,pussy,penis）
-e RATIO           検知範囲を bbox サイズの RATIO 倍拡張（例: 0.1 = 10% 拡張）
-rm                高解像度セグメンテーションマスクを使用
-sp                検出枠つきプレビュー画像を追加出力
-sm                マスク画像を追加出力
-n, --no-meta      メタデータのコピーを無効化（問題発生時に試す）
-d DEVICE          処理デバイス（CPU 指定: -d cpu）
```
